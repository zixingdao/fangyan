# 长沙方言采集平台 - 项目初始化计划

根据提供的《项目文档》，我将为您搭建完善的前后端分离项目框架。

## 1. 项目结构初始化
- 建立符合文档规范的目录结构：
  - `frontend/`: 前端项目
  - `backend/`: 后端项目
  - `database/`: 数据库脚本
  - `scripts/`: 运维脚本

## 2. 前端框架搭建 (frontend)
- **技术栈**: Vue 3 + TypeScript + Vite
- **UI框架**: Element Plus (按需引入)
- **状态管理**: Pinia
- **路由**: Vue Router 4
- **HTTP客户端**: Axios (封装请求拦截器)
- **CSS预处理**: Sass
- **目录结构创建**:
  - `src/layouts`: 实现 UserLayout (前台) 和 AdminLayout (后台)
  - `src/views`: 创建 web (主页、登录) 和 admin (仪表盘) 的基础页面占位
  - `src/api`: 创建 API 模块结构
  - `src/store`: 初始化 Pinia
- **配置**: 配置 Vite 代理解决开发环境跨域问题

## 3. 后端框架搭建 (backend)
- **技术栈**: Node.js + Express + TypeScript
- **基础依赖**: `cors`, `dotenv`, `helmet`, `morgan`
- **开发工具**: `nodemon`, `ts-node`
- **目录结构创建**:
  - `src/app.ts`: 应用入口，配置中间件
  - `src/routes`: 路由分层 (api/v1)
  - `src/controllers`: 控制器层
  - `src/services`: 业务逻辑层
  - `src/models`: 数据模型定义
  - `src/config`: 环境变量配置
- **功能实现**: 实现一个基础的健康检查接口 `/api/health`

## 4. 数据库准备 (database)
- 创建 `database/init.sql`: 写入文档中定义的 `users`, `recordings`, `rankings`, `password_reset_requests` 四张表的建表语句。

## 5. 验证与运行
- 在根目录创建 `README.md` 说明启动方式。
- 确保前端可以启动并显示主页。
- 确保后端可以启动并响应 API 请求。

准备好后，我将开始执行上述步骤。
